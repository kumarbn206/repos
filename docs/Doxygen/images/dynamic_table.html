RFE dynamic table blob structure

The RFE dynamic table blob (binary large object) can be used to define a limited set of configuration values that will be updated during chirp generation by the RFE software.

The image below overlays an example RFE dynamic table blob with the corresponding high-level structure:

Global constants section

The global constants section contains the number of metatables present in the RFE dynamic table blob and its total size. This section is structured according to the table below:

Byte index: Parameter: Description:
0 Table count Number of metatables present in the RFE dynamic table blob.
1 <reserved value>
2 Total size The total size of the RFE dynamic table blob.
3

Metatable section

The metatable section describes the contents of a single dynamic table. These sections are structured according to the table below:

Byte index: Parameter: Description:
0 Data table offset The offset to the start of the data table present in the data section.
1
2 Data table size The size of the data table present in the data section.
3
4 Sequence length The length of each sequence present in the data table present in the data section.
5
6 <reserved value>
7
8 Chirp profile index sequence offset The offset to the start of the chirp profile index sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
9
10 Phase rotation TX 1 sequence offset The offset to the start of the phase rotation TX 1 sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
11
12 Phase rotation TX 2 sequence offset The offset to the start of the phase rotation TX 2 sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
13
14 Phase rotation TX 3 sequence offset The offset to the start of the phase rotation TX 3 sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
15
16 Phase rotation TX 4 sequence offset The offset to the start of the phase rotation TX 4 sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
17
18 TX transmission enable sequence offset The offset to the start of the TX transmission enable sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
19
20 Dwell time sequence offset The offset to the start of the dwell time sequence present in the data section. The absence of this specific sequence is indicated by an offset value of 0x0000.
21

Data section

The data section contains all sequences defined for this RFE dynamic table blob.

The precise memory range occupied by a sequence is defined by three properties:

  • The offset is determined by the corresponding offset value in the corresponding metatable.
  • The size of an element is dependent on the type of the sequence and is 1, 2, or 4 bytes.
  • The number of elements is determined by the sequence length value in the corresponding metatable.
